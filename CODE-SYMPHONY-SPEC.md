# Code Symphony - GitHub 기여도를 음악으로

> 당신의 코딩 여정을 들어보세요

---

## 1. 프로젝트 개요

### 컨셉
GitHub 사용자의 기여도(contribution) 데이터를 분석하여 고유한 음악을 실시간으로 생성하는 웹 애플리케이션.

### 왜 만드나?
- **재미**: 코딩 활동을 음악으로 듣는 신선한 경험
- **바이럴 잠재력**: "내 GitHub 소리" 공유 욕구 자극
- **GitHub 스타 획득**: 개발자 커뮤니티 타겟 → 스타 전환율 높음

### 목표
- GitHub 스타 500개 (6개월 내)
- Product Hunt 데일리 Top 10
- Twitter/X 개발자 커뮤니티에서 바이럴

### 레퍼런스
- [gh-space-shooter](https://github.com/czl9707/gh-space-shooter) - GitHub 기여도 → 슈팅 게임
- [GitHub Skyline](https://skyline.github.com/) - GitHub 기여도 → 3D 도시

## 2. 핵심 기능

### 2.1 기여도 → 음악 변환
```
GitHub Username 입력 → 기여도 데이터 수집 → 음악 실시간 생성 → 재생
```

### 2.2 기능 목록

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 음악 생성 | 기여도 기반 멜로디 실시간 생성 | P0 (필수) |
| 시각화 | 음악에 맞춰 움직이는 기여도 그래프 | P0 (필수) |
| 공유 | Twitter/X 공유 버튼 + OG 이미지 | P0 (필수) |
| 다운로드 | MP3/WAV 파일 다운로드 | P1 (권장) |
| 테마 | 장르 선택 (클래식, 일렉트로닉, 로파이) | P2 (선택) |
| 비교 | 두 사용자 음악 동시 재생 | P2 (선택) |

### 2.3 사용자 플로우

```
[랜딩 페이지]
     │
     ▼
[GitHub 유저네임 입력]
     │
     ▼
[로딩 애니메이션] ← "당신의 코드를 연주 중..."
     │
     ▼
[음악 재생 + 시각화]
     │
     ├─→ [Twitter 공유] → 바이럴 루프
     ├─→ [MP3 다운로드]
     └─→ [다른 유저 검색]
```

## 4. 데이터 → 음악 매핑 규칙

### 4.1 GitHub 데이터 수집

```graphql
query($username: String!) {
  user(login: $username) {
    contributionsCollection {
      contributionCalendar {
        weeks {
          contributionDays {
            date
            contributionCount
          }
        }
      }
    }
  }
}
```

### 4.2 음악 변환 규칙

#### 스케일: C Major Pentatonic (불협화음 방지)
```
C4 - D4 - E4 - G4 - A4 - C5 - D5 - E5
```

#### 매핑 테이블

| 기여도 데이터 | 음악 요소 | 변환 규칙 |
|--------------|----------|----------|
| 기여 횟수 (일별) | 음의 높이 | 0=C4, 1-3=D4, 4-7=E4, 8-12=G4, 13+=A4 |
| 기여 횟수 (일별) | 음의 길이 | 0=쉼표, 1-5=8분음표, 6+=4분음표 |
| 연속 기여 일수 | 코드 진행 | streak ≥7일 → 코드 추가 (C-G-Am-F) |
| 요일 | 베이스 음 | 월=C2, 화=D2, 수=E2, 목=G2, 금=A2, 주말=쉼표 |
| 주간 총 기여 | 템포 | 0-10=60bpm, 11-30=80bpm, 31+=100bpm |

#### 예시 변환

```
월요일: 5 commits  → D4 (8분음표) + C2 베이스
화요일: 12 commits → G4 (4분음표) + D2 베이스
수요일: 0 commits  → 쉼표
목요일: 3 commits  → D4 (8분음표) + G2 베이스
금요일: 8 commits  → G4 (4분음표) + A2 베이스
```

### 4.3 음악 구조

```
[인트로 4마디] → [52주 기여도 연주] → [아웃트로 4마디]
     │                  │                    │
   총 기여도 기반     주별 루프 (52회)      페이드 아웃
   코드 진행          일별 멜로디 생성
```
